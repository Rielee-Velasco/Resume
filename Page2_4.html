Lost or Stolen Process for PC

Step 1: Security incident ticket creation

  1. Document the relevant details of theft or loss incident including the user's contact number.
  2. Report the incident to the Security Team following the instructions in KBA0000 Security - What can be a security incident and how to report it to the Security Operations Center.

Note: Even when any information required by the Security Team couldn't be obtained from the user, it is essential that the incident is reported to them immediately. Any missing information will be provided by the local Service Desk as soon as possible.

Step 2: Information to the user

Global service desk will inform the user of the following:

  1. For security reasons, the AD password/XYZ Authentication code will be reset, making it impossible to access the Company ABC environment. Additionally, the 123 key will be revoked if the user has one.
  2. The Local Service Desk will contact the user during business hours (8:30 EST) to gather more information and draft a "Stolen/Lost Report" for internal reporting. This report will be sent to the user and their line manager for completion.
  3. The user must report the incident to the police and obtain a copy of the "theft report" or "lost property report."
  4. The user should submit the completed "Stolen/Lost Report" and the copy of the police report to the local Service Desk as soon as possible. The new device request process can only be started once the "Stolen/Lost Report" is submitted.
  5. Even if the device is found, the "Stolen/Lost Report" must still be submitted, and the device must be returned to IT for security inspection. A new IT device request is necessary in this case as well.
  6. The same information as above will be sent to the user's line manager by email.

Step 3: Email communication to the user and user's line manager

Global service desk agent also emails the same information as above to the user and user's line manager by using email template in the following link.

Step 4: XYZ/eToken Revocation

Global service desk agent revokes XYZ/eToken using XYZ admin tool.

Step 5: EFG Key Revocation

Global service desk agent raises a EFG Key revoke request to EFG Support team by e-mail.

Step 6: Creation of another incident and hand-over to local Service Desk

  1. GlobalService Desk will create another incident ticket documenting the following:
    • All the relevant information about the theft or loss provided by the user:
    • User's contact number:
    • Actions taken:
    • Security ticket number:
    • Was the user provided the information indicated in Step 2?
    • Was the email communication sent to the user and user's line manager?
    • Was the XYZ/eToken revocation completed?
    • Was the email sent to EFG Support team team to request revocation of EFG team key?
  2. Global Service Desk will send an e-mail to local Service Desk to inform the ticket assignment and the incident ticket number.
  3. Local Service Desk will take over the Incident ticket and will complete the remaining process.
